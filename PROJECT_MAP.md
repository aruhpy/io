# PROJECT_MAP.md (myblog / Hugo + GitHub Pages /io/)

> 이 파일은 ChatGPT가 새 채팅에서도 **정확한 파일 경로로 안내**하도록 하기 위한 "프로젝트 지도"입니다.  
> 새 채팅 시작 시: 이 파일 + (필요한 경우) single.html / baseof.html / components.css를 함께 첨부합니다.

---

## 0) 프로젝트 목표 / 현재 진행상태

### 현재 완료된 것(중요)
- 타입 상세 페이지에서 **샘플 이미지 영역은 ‘가로 슬라이드(스크롤 스냅)’ + ‘라이트박스(클릭 확대)’**로 정상 작동.
- PC/모바일 모두 동작하며, 모바일 UX는 만족(좌우 스와이프 + 탭 확대 OK).
- 가격 영역은 프론트매터 `price.rows`를 출력하고 있으며,
  **"점점점 메뉴판(… … …)" 스타일** 디자인을 적용해둔 상태.

### 앞으로 작업할 것(우선순위)
1) `price.rows` 출력 UI에서
   - "위+아래 주민 1.5"
   - "옵션: 위 주민+아래 크리쳐 (+0.5)"
   이런 문장을 **가독성 좋은 디자인**으로 바꾸기 (예시/디자인 방향 먼저 결정)
2) 그 이후에 타입 상세의 나머지 디자인(신청양식/주의사항/불가주민 아코디언+검색+칩 등)을 단계적으로 확장 예정

### 작업 방식(매우 중요)
- 앞으로는 **한 번에 여러 군데를 건드리지 않고**,
  사용자가 요청한 “한 가지”만 **차근차근** 변경한다.
- CSS는 기존 규칙을 찾아 **교체 방식**으로 제공한다(덮어쓰기 추가 방식 지양).
- 코드 제공 시 `...` 생략 없이 **통째 복붙 가능한 전체 블록**으로 제공한다.

## 1) 핵심 경로(진짜 자주 수정하는 파일)

### (A) 타입 상세 템플릿 (가장 중요)
- `layouts/type/single.html`
  - 타입 상세 페이지 레이아웃/출력 로직
  - 샘플 슬라이더 마크업
  - 라이트박스 마크업
  - 프론트매터(예: price.rows, duplicate_notice, ban_residents 등) 출력 처리

> ⚠️ single.html이 다른 위치에도 있을 수 있으나,
> "타입 상세(type/single) 커스터마이징"은 위 파일이 기준입니다.

### (B) 공통 베이스 레이아웃
- `themes/naru/layouts/_default/baseof.html`
  - 전체 HTML 골격
  - 홈버튼/프레임(canvas_frame_v2.svg, mobile_frame.svg)
  - 이미지 우클릭 방지/드래그 방지/터치 관련 스크립트
  - 타입 상세 라이트박스/슬라이더와 충돌 가능 스크립트가 있으면 여기서 조정

### (C) 컴포넌트 CSS (공지/타입정리/타입상세)
- `themes/naru/static/css/components.css`
  - 타입정리 페이지 그리드/칩바/캡션
  - 타입상세 슬라이더(.type-gallery-slider)
  - 타입상세 라이트박스(.type-lightbox)
  - 타입상세 가격 영역(점점점 메뉴판 스타일)
  - 공지사항 컴포넌트(필독/아코디언 등)

### (D) 스타일 엔트리/레이아웃/진행상황
- `themes/naru/static/style.css`  (엔트리: @import 모음)
- `themes/naru/static/css/layout.css` (레이아웃/프레임/반응형/홈버튼)
- `themes/naru/static/css/progress.css` (진행상황 전용)

  - 샘플 이미지 파일들(여기에 이미지 넣으면 템플릿이 자동 렌더)

## 3) 프론트매터 규칙(중요)

### 공통
- 각 파일당 `---`는 시작/끝 **1번씩만**
- 리스트는 반드시
  - `- "이름"` 형태로 **한 줄씩**

### price 구조(권장)
- 문장형으로 쓰지 말고 `rows:` 구조로 유지
- 예시:
  price:
    rows:
      - label: "위 + 아래 주민"
        value: "1.5"
      - label: "옵션: 위 주민 + 아래 크리쳐"
        value: "+0.5"
      - label: "작업 범위"
        value: "위+아래 주민 / 위 주민+아래 크리쳐"
